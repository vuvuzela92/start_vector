# Перенос поквартального плана в свод по поставщикам

## Назначение

Скрипт переносит данные из Google-таблицы `Годовой план закупа 2026`
в Google-таблицу `Расчет поставки Китай_по обороту`.

Источник данных - лист `quarter_sheet`.
Целевой лист - `db_sheet_quarterly`.

При записи используется общий метод проекта `GoogleTabs.set_df_to_google()`.
Он перезаписывает данные на целевом листе и добавляет колонку `updated_at`
с датой и временем обновления.

## Входные данные

Входные данные читаются из Google Sheets:

- таблица-источник: `annual_procurement_plan["title"]`;
- лист-источник: `annual_procurement_plan["quarter_sheet"]`;
- строка заголовков: 4-я строка листа;
- строки данных: начиная с 5-й строки.

Заголовки берутся из исходного листа, после чего DataFrame приводится
к фиксированной структуре колонок из `ANNUAL_PLAN_COLUMNS`.

## Структура колонок

Итоговый DataFrame приводится к списку `ANNUAL_PLAN_COLUMNS`
из `src_oop/jobs/calculation_of_purchases_china/config.py`.

Правила формирования:

- порядок колонок всегда соответствует `ANNUAL_PLAN_COLUMNS`;
- если нужной колонки нет в исходном листе, она добавляется с пустыми значениями;
- колонки, которых нет в `ANNUAL_PLAN_COLUMNS`, не передаются в целевой лист;
- колонка `updated_at` добавляется отдельно методом `set_df_to_google()` при записи.

## Выходные данные

Данные записываются в Google Sheets:

- таблица-приемник: `delivery_calculation_china["title"]`;
- лист-приемник: `delivery_calculation_china["db_sheet_quarterly"]`.

В целевой лист записываются:

- колонки из `ANNUAL_PLAN_COLUMNS` в заданном порядке;
- дополнительная колонка `updated_at`, которую добавляет `set_df_to_google()`.

## Порядок запуска

Запуск через общий CLI проекта:

```powershell
python main.py transport_quarterly_plan_to_pivot
```

Функция запуска находится в `run.py`:

```python
transport_quarterly_plan_to_pivot()
```

## Настройки

Используются настройки из двух файлов:

- `src_oop/jobs/annual_procurement_plan/config.py`
- `src_oop/jobs/calculation_of_purchases_china/config.py`

Из `annual_procurement_plan/config.py` используются:

- `annual_procurement_plan["title"]` - название исходной таблицы;
- `annual_procurement_plan["quarter_sheet"]` - название исходного листа.

Из `calculation_of_purchases_china/config.py` используются:

- `delivery_calculation_china["title"]` - название целевой таблицы;
- `delivery_calculation_china["db_sheet_quarterly"]` - название целевого листа;
- `ANNUAL_PLAN_COLUMNS` - фиксированный список колонок итогового DataFrame.

Для подключения к Google Sheets используется `GoogleTabs`.
Он читает сервисный аккаунт из:

```text
creds/creds.json
```

Файл с доступами должен существовать локально и не должен попадать в git.

## Возможные ошибки

### Не найдена Google-таблица

Проверьте:

- название таблицы в `config.py`;
- доступ сервисного аккаунта к таблице;
- файл `creds/creds.json`.

### Не найден лист

Проверьте:

- `annual_procurement_plan["quarter_sheet"]`;
- `delivery_calculation_china["db_sheet_quarterly"]`;
- наличие листов в соответствующих Google-таблицах.

### Исходный лист пустой или нет заголовков

Скрипт ожидает:

- заголовки в 4-й строке;
- данные начиная с 5-й строки.

Если строк меньше четырех, обновление целевого листа не выполняется.

### В целевом листе не хватает колонок

Проверьте список `ANNUAL_PLAN_COLUMNS` в `config.py`.
Если колонка отсутствует в этом списке, скрипт не будет передавать ее в целевой лист.

### Целевой лист не обновился

Проверьте:

- что команда запуска завершилась без traceback;
- что в целевом листе появилась или обновилась колонка `updated_at`;
- что сервисный аккаунт имеет права на редактирование целевой таблицы.

## Проверка результата

После запуска откройте целевой лист `db_sheet_quarterly` и проверьте:

- появились данные из исходного листа;
- порядок колонок соответствует `ANNUAL_PLAN_COLUMNS`;
- отсутствующие в исходнике колонки заполнены пустыми значениями;
- есть колонка `updated_at`;
- значение `updated_at` соответствует времени последнего запуска.
