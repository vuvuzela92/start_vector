# AGENTS.md

## Правила для `src_oop/tasks_registry.py`

- Не удаляй существующие комментарии, разделители и секции-группировки в `src_oop/tasks_registry.py`.
- Сохраняй структуру реестра задач по бизнес-блокам, чтобы по файлу было легко ориентироваться.
- При добавлении новой задачи вставляй её в подходящий смысловой раздел, а не просто в конец списка.
- Если комментарии в `src_oop/tasks_registry.py` устарели, не удаляй их молча: обнови или аккуратно восстанови навигационные комментарии.
- Нельзя превращать `src_oop/tasks_registry.py` в сплошной список задач без секций и поясняющих комментариев.
- Любые изменения в `src_oop/tasks_registry.py` должны сохранять или улучшать существующую систему комментариев-навигации.
- Удаление комментариев в `src_oop/tasks_registry.py` запрещено без явного запроса пользователя.
